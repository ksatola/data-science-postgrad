Dostęp do Amazon EC2 - Instrukcja

Informacje podstawowe:

1.     Strona główna Amazon Web Services: https://aws.amazon.com/

2.     Rejestracja w Education Program: formularz dla studentów.

3.     Dodatkowy kupon z GitHub: Student Developer Pack Uwaga: z kuponu z GitHub należy skorzystać przed zgłoszeniem się do AWS Education Program.

4.     Założenie konta: https://portal.aws.amazon.com/gp/aws/developer/registration/index.html

5.     Doładowanie konta przy użyciu kuponu: https://console.aws.amazon.com/billing/home?#/credits

6.     Sprawdzenie stanu konta: https://console.aws.amazon.com/billing/home?#/

7.     Uwaga: przekroczenie kwoty doładowania powoduje obciążenie karty! Studenci korzystają z platformy Amazon EC2 na własną odpowiedzialność i odpowiadają za wszelkie koszty ponad wartość przydzielonego kuponu.

8.     Uwaga Należy wybrać opcję podstawowego wsparcia (basic support) - inne opcje są płatne.

 

Dostęp do Elastic Compute Cloud (EC2) poprzez konsolę webową:

1.     Konsola EC2: https://console.aws.amazon.com/ec2/v2/home?region=eu-west-1

2.     Najlepiej wybrać region EU (Ireland) (Uwaga: od niedawna na koncie studenckim może być możliwość startowania instancji tylko w regionie us-east-1)

3.     Uruchomienie testowej instancji:

  1. Obraz maszyny wirtualnej (Amazon Machine Image AMI) Ubuntu Server 16.04 LTS (HVM), SSD Volume Type - ami-0ac8ad8d8155dd79d 
  2. Rozmiar instancji: ﻿t2.micro
  3. Konfiguracja bezpieczeństwa: port 80 (HTTP) musi być otwarty, opcjonalnie port 4040
  4. (Opcjonalne) Utworzenie nowej pary kluczy i zapisanie pliku klucz.pem.

4.     Połączenie do uruchomionej instancji:

  • Połączenie przez przeglądarkę, przykładowy adres: http://c2-176-34-70-221.eu-west-1.compute.amazonaws.com
  • Zapisanie obrazu instancji: Actions -> Create Image

 

Uwaga: na zajęciach korzystamy z obrazu: ami-0ac8ad8d8155dd79d (region eu-west-1) lub ami-010505bc5a5fd54cb (region us-east-1 N. Virgnia)
